# Руководящие принципы использования дизайн-системы

## Общие правила

### Именование
- Используйте единый нейминг для всех компонентов
- Следуйте конвенциям проекта (camelCase для переменных, PascalCase для компонентов)
- Используйте описательные имена, отражающие назначение компонента

### Импорты
Всегда импортируйте компоненты из дизайн-системы:

```tsx
// ✅ Правильно
import { Button, Switch, Label } from 'src/design-system/components';
import { Table, TableContainer, TableHead, TableBody, TableRow, TableCell } from 'src/design-system/components';

// ❌ Неправильно
import { Button, Table, TableCell } from '@mui/material';
```

### Переиспользование
- Используйте компоненты дизайн-системы вместо создания новых
- Если нужна кастомизация, расширяйте существующие компоненты через props
- Избегайте дублирования кода

## Правила использования компонентов

### Button

1. **Выбор варианта:**
   - `contained` - для главных действий
   - `outlined` - для вторичных действий
   - `text` - для третичных действий
   - `gradient` - для акцентных действий
   - `surface` - для действий на тёмном фоне

2. **Размеры:**
   - `large` - для основных CTA (Call-to-Action)
   - `medium` - для стандартных действий
   - `small` - для компактных интерфейсов

3. **Состояния:**
   - Всегда используйте `loading` prop для состояния загрузки
   - Не создавайте кастомные индикаторы загрузки
   - Используйте `disabled` для недоступных действий

### Switch

1. **Использование:**
   - Для включения/выключения опций
   - Для переключения между двумя состояниями
   - Всегда с меткой (label)

2. **Размеры:**
   - `medium` - стандартное использование
   - `small` - для компактных интерфейсов

3. **Состояния:**
   - Используйте `disabled` для недоступных опций
   - Для loading состояния используйте обёртку с индикатором

### Table

1. **Импорт:** `Table`, `TableContainer`, `TableHead`, `TableBody`, `TableRow`, `TableCell`, `tableCellClasses` из `src/design-system/components`.
2. **Структура:** `TableContainer` → `Table` → `TableHead`/`TableBody` → `TableRow` → `TableCell`.
3. **Заголовки:** в ячейках `TableHead` используйте `Typography variant="helperText"`.
4. **Прокрутка:** при `maxHeight` у `TableContainer` используйте `Table` с `stickyHeader` для фиксированного заголовка.

### Label

1. **Импорт:** `Label` из `src/design-system/components`.
2. **Использование:** метки над полями ввода и селектами. Для Switch/Checkbox — `FormControlLabel`.
3. **Доступность:** задавайте `htmlFor` и `id` у поля для связи с меткой.

## Цветовая палитра

### Использование цветов
- Используйте семантические цвета (error, warning, success, info) по назначению
- Не используйте цвета напрямую, всегда через тему
- Поддерживайте светлую и тёмную темы

### Примеры:
```tsx
// ✅ Правильно
<Typography color="error.main">Ошибка</Typography>
<Button color="success">Успех</Button>

// ❌ Неправильно
<Typography sx={{ color: '#F44336' }}>Ошибка</Typography>
```

## Типографика

### Выбор варианта
- Используйте правильный вариант для контекста
- Не смешивайте размеры без необходимости
- Следуйте иерархии заголовков

### Примеры:
```tsx
// ✅ Правильно
<Typography variant="h1">Главный заголовок</Typography>
<Typography variant="description">Основной текст</Typography>

// ❌ Неправильно
<Typography variant="h1" sx={{ fontSize: 14 }}>Текст</Typography>
```

## Масштабирование

### Добавление новых компонентов
1. Создайте папку компонента в `src/design-system/components/`
2. Создайте файл компонента с типизацией
3. Создайте README.md с полной документацией
4. Добавьте примеры использования
5. Обновите главный README.md

### Расширение существующих компонентов
1. Добавьте новые props с типизацией
2. Обновите документацию
3. Добавьте примеры использования новых возможностей
4. Убедитесь в обратной совместимости

## Тестирование

### Компоненты должны:
- Работать в светлой и тёмной темах
- Поддерживать все заявленные состояния
- Быть доступными (a11y)
- Иметь правильную типизацию

### Примеры тестов:
```tsx
// Проверка состояний
it('should be disabled when loading', () => {
  render(<Button loading>Test</Button>);
  expect(screen.getByRole('button')).toBeDisabled();
});

// Проверка тем
it('should support dark mode', () => {
  const { container } = render(
    <ThemeProvider theme={darkTheme}>
      <Button>Test</Button>
    </ThemeProvider>
  );
  // assertions
});
```

## Документирование

### Каждый компонент должен иметь:
1. README.md с описанием
2. Примеры всех вариантов и состояний
3. Описание ограничений
4. Примеры использования
5. Информацию о доступности
